<h3 align="left">📱 QweR – KoBART 기반 QR 코드 요약 앱</h3>
<p><strong>역할:</strong> Android Developer, NLP Model Trainer</p>
<p><strong>설명:</strong><br>
정보 과잉 시대에 사용자가 QR 코드로 접속한 문서를 빠르게 요약하고, 유해 콘텐츠를 차단할 수 있는 모바일 애플리케이션 <strong>‘QweR’</strong>를 개발했습니다. 
안드로이드 프론트엔드와 KoBART 요약 모델 학습을 담당했습니다.
</p>

<ul>
  <li><strong>S:</strong> NLP 경험이 없는 상태에서 KoBART 모델의 요약 품질과 속도를 동시에 개선해야 했습니다.</li>
  <li><strong>T:</strong> 기사 데이터를 직접 수집해 KoBART 모델을 학습하고, 속도 문제를 해결하기 위해 Batch Size 조정으로 처리 시간을 <strong>15초 → 10초</strong>로 단축했습니다.</li>
  <li><strong>A:</strong> ChatGPT API를 대안 모델로 연동하여, 사용자가 두 엔진 중 선택 가능하도록 구성했습니다. Kotlin 기반 Android 앱에서 MVVM 패턴을 적용하여 구조적 유지보수성과 실시간 요약 반영을 구현했습니다.</li>
  <li><strong>R:</strong> KoBART와 GPT를 통합한 하이브리드 요약 시스템으로 정확도·속도 균형을 달성, <strong>우수 프로젝트로 선정</strong>되었습니다.</li>
  <li><strong>L:</strong> NLP 파이프라인 및 API 연동 구조를 익히고, 실험 데이터 기반의 의사결정과 사용자 중심 성능 개선 경험을 쌓았습니다.</li>
</ul>

<h3 align="left">📊 Candly – AI 기반 주식 차트 학습 서비스</h3>
<p><strong>역할:</strong> PM, Fullstack Developer (React + Node.js)</p>
<p><strong>설명:</strong><br>
초보 투자자들이 실제 데이터를 기반으로 차트 해석과 재무 분석을 학습할 수 있도록 하는 웹 서비스 <strong>‘Candly’</strong>를 기획·개발했습니다.
</p>

<ul>
  <li><strong>S:</strong> 20년치 주식·재무 데이터를 가공해 학습용 차트를 제공해야 했습니다.</li>
  <li><strong>T:</strong> PER·PBR·ROE 등 주요 지표 자동 계산 및 사용자별 학습 이력 시각화가 핵심 과제였습니다.</li>
  <li><strong>A:</strong> Dart OpenAPI로 KOSPI200 종목의 20년치 데이터를 수집하고, Python+Node.js로 전처리했습니다. React 기반 클라이언트에서 성장성/수익성 복합 그래프를 구현하고, 마이페이지에는 잔디형 캘린더를 적용했습니다.</li>
  <li><strong>R:</strong> 데이터 로딩 속도 <strong>35% 개선</strong>, 차트 정확도 <strong>99%</strong> 유지, 4주 내 완성도로 <strong>팀 최고 완성도 프로젝트 선정</strong>.</li>
  <li><strong>L:</strong> API 파이프라인, 복합형 차트 렌더링, UX 중심의 데이터 시각화 설계를 익혔습니다. 이후 <strong>SOL Assistant</strong> 개발 시 투자자 행동 분석으로 확장되었습니다.</li>
</ul>

<h3 align="left">💹 SOL Assistant – 초보자 투자 어시스트 대시보드</h3>
<p><strong>역할:</strong> PM, Backend Developer</p>
<p><strong>설명:</strong><br>
신한투자증권 프로디지털 아카데미 파이널 프로젝트로, 초보 투자자를 위한 <strong>투자 어시스트 대시보드 ‘SOL Assistant’</strong>를 개발했습니다. 
사용자의 투자 성향과 실시간 시장 데이터를 분석해 맞춤형 리포트를 제공하는 인앱 서비스로, 확장성과 실시간성 확보를 목표로 했습니다.
</p>

<ul>
  <li><strong>S:</strong> 실시간 데이터 처리와 서비스 확장을 위해 단일 구조를 MSA로 전환해야 했습니다.</li>
  <li><strong>T:</strong> 서비스 간 결합도를 낮추면서 대용량 데이터를 안정적으로 교환하는 시스템을 설계했습니다.</li>
  <li><strong>A:</strong> 
    <ul>
      <li><strong>MSA 기반 4개 모듈 설계:</strong> internal, external, chart-similarity, common-service로 분리</li>
      <li><strong>행동 데이터 파이프라인 구현:</strong> Redis → Kafka → RDS 비동기 구조로 초당 수천 건 이벤트 처리</li>
      <li><strong>PM 역할:</strong> 아키텍처 정의, API 규격서 및 Kafka 토픽 설계 주도</li>
    </ul>
  </li>
  <li><strong>R:</strong> Kafka 기반 비동기 처리로 통신 지연을 <strong>300ms → 40ms</strong>로 단축, Redis 캐싱으로 조회 속도 <strong>45% 개선</strong>. <strong>최우수 프로젝트 선정</strong>.</li>
  <li><strong>L:</strong> 
    <ul>
      <li>Kafka·Redis 등 <strong>분산 시스템 아키텍처 설계 역량</strong> 확보</li>
      <li>팀 리더로서 <strong>데이터 흐름 조율·문제 복구 프로세스 설계</strong> 경험</li>
    </ul>
  </li>
</ul>
